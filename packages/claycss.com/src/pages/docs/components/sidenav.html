To create a sidenav relative to other elements inside your page, you must follow the markup structure shown above and pass the sidenav-container element to the plugin with the {literal}href{/literal} attribute or the {literal}data-target{/literal} attribute on the toggler. The {literal}container{/literal} option also allows you to pass the container via javascript if {literal}href{/literal} or {literal}data-target{/literal} can't be used. The options above are optional.

Avoid the flash of unstyled content when using the Javascript api by adding the class sidenav-js-fouc to the sidenav-container.

Side Navigation Data API

The Side Navigation data api was added for situations where there is limited control over the markup structure. The only markup needed is a toggler and a panel to hide and show. These items can be included anywhere on the page.

The minimum amount of attributes required is {literal}data-toggle="sidenav"{/literal} and {literal}href="#target"{/literal} or {literal}data-target="#target"{/literal}. This will open and close a fixed position sidenav provided the markup below. The width of the {literal}sidenav-slider{/literal}, {literal}sidenav-menu{/literal}, and position of the sidenav must be set through css. When using {literal}fixed-push{/literal}, padding must also be set via css on open to push the content over.

The data api allows for fixed-push style sidenavs with the attributes {literal}data-content{/literal}, {literal}data-type{/literal}, and {literal}data-type-mobile{/literal}. {literal}Data-content{/literal} is the id of the content to be pushed over when the nav is open. {literal}Data-type{/literal} and {literal}data-type-mobile{/literal} give the option to specify the style of sidenav in desktop and mobile, respectively. The values can be {literal}fixed{/literal} or {literal}fixed-push{/literal} and defaults to fixed.

The open class name can be customized through the attribute {literal}data-open-class{/literal} and will be added to toggler and content on open.

Lazy Loading Side Navigation Content

You can load content into the sidebar of the Side Navigation in one of two ways. At the very minimum, the only two things you need are a URL and that the sidebar has a child element with the {literal}sidebar-body{/literal} class. Data fetched from the URL will be appended to the {literal}sidebar-body{/literal} element.

Using the Data API

Use attribute {literal}data-url{/literal} on the toggle to load content into {literal}.sidenav-body{/literal} when the side navigation is opened.

Lazy Loading Events and State

When using the {literal}url{/literal} or {literal}data-url{/literal} options, the plugin will fire an event when the content is loaded {literal}urlLoaded.lexicon.sidenav{/literal}.

Also, while the content is being loaded, an element {literal}.sidebar-loading{/literal} containing a loading animation will be added to the {literal}.sidebar-body{/literal} element. The {literal}.sidebar-loading{/literal} element will be removed when the content is done loading.

The loading animation defaults to dotted spinner but can be customized through {literal}loadingIndicatorTPL{/literal} in the options API or {literal}data-loading-indicator-tpl{/literal} in the data API.

If using the options API, the class will be added to the {literal}.sidenav-menu{/literal} element.

Events

{literal}openStart.lexicon.sidenav{/literal}

Fires when the side navigation begins to open.

{literal}open.lexicon.sidenav{/literal}

Fires when the side navigation is fully opened.

{literal}closedStart.lexicon.sidenav{/literal}

Fires when the side navigation begins to close.

{literal}closed.lexicon.sidenav{/literal}

Fires when the side navigation is fully closed.

Methods

{literal}.sideNavigation('toggle'){/literal}

Manually toggles the sidenav. Be aware that this method will return to the caller before the sidenav has been shown or hidden.

{call ElectricCode.render} {param code kind="text"} {literal}$('#myTogglerId').sideNavigation('toggle'){/literal} {/param} {param mode: 'xml' /} {/call}

{literal}.sideNavigation('show'){/literal}

Manually shows the sidenav. Be aware that this method will return to the caller before the sidenav has been shown.

{call ElectricCode.render} {param code kind="text"} {literal}$('#myTogglerId').sideNavigation('show'){/literal} {/param} {param mode: 'xml' /} {/call}

{literal}.sideNavigation('hide'){/literal}

Manually toggles the sidenav. Be aware that this method will return to the caller before the sidenav has been hidden.

{call ElectricCode.render} {param code kind="text"} {literal}$('#myTogglerId').sideNavigation('hide'){/literal} {/param} {param mode: 'xml' /} {/call}

{literal}.sideNavigation('destroy'){/literal}

Destroys the Side Navigation instance for the items in the jQuery collection, and removes any attached event handlers and any associated data added by the plugin.

{call ElectricCode.render} {param code kind="text"} {literal}$('#myTogglerId').sideNavigation('destroy'){/literal} {/param} {param mode: 'xml' /} {/call}

{literal}.sideNavigation('instance'){/literal}

Returns the Side Navigation instance for the first item in the jQuery collection (similar to how any other jQuery getter method works). If the plugin hasn't been initialized, it will return null.

{call ElectricCode.render} {param code kind="text"} {literal}$('#myTogglerId').sideNavigation('instance'){/literal} {/param} {param mode: 'xml' /} {/call}

{literal}.sideNavigation('visible'){/literal}

Returns the visible state of the sidenav.

If the sidenav is in the middle of transitioning, this will report what the visible state will be after the transition is finished. This allows you to call it in any state and accurately get the state of the sidenav.

{call ElectricCode.render} {param code kind="text"} {literal}$('#myTogglerId').sideNavigation('show').sideNavigation('visible'); // returns true $('#myTogglerId').sideNavigation('hide').sideNavigation('visible'); // returns false{/literal} {/param} {param mode: 'xml' /} {/call}
